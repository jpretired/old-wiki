Translating relationships

From webtrees
Jump to: navigation, search

Translating relationships is difficult. We cannot simply translate the names, as the semantics are different. One person's "3rd cousin twice removed" is another person's "cousin of the 7th degree".

The existing code creates a translation for every relationship that exists in any language - and then asks all languages to translate all relationships. This requires translations for terms that do not exist. For example, the Spanish have a word "consuegro" (son's wife's father) which cannot be translated into other languages.

The proposed solution to this is to create separate rules and translations for each language. These require an understanding of regular expressions and mysql expressions, and of course, the target language.

How it works

Complex relationships are created from 4 basic relationship types (parent, child, sibling, spouse), and 3 sexes (male, female, unknown).

father son brother husband
mother daughter sister wife
parent child sibling spouse

These are abbreviated to the first 3 letters.

fat son bro hus
mot dau sis wif
par chi sib spo

A relationship is defined using the shortest path between two individuals.

For example:

  • bro = brother
  • fatson = half-brother
  • fatwifson = step-brother

Some relationships can be defined exactly. For example:

  • fatsis = 'aunt'
  • motsis = 'aunt'
  • parsis = 'aunt'
  • broson = 'nephew'
  • sisson = 'nephew'
  • sibson = 'nephew'

But to save lots of repetition, we can use a type of pattern matching called regular expressions. These relationships can therefore be defined more compactly like this:

  • (fat|mot|par)sis = 'aunt'
  • (bro|sis|sib)son = 'nephew'

We also need a way to record repeating relationships. For example:

  • fat = 'father'
  • (fat|mot|par)fat = 'grandfather'
  • (fat|mot|par)(fat|mot|par)fat = 'great-grandfather'
  • (fat|mot|par)(fat|mot|par)(fat|mot|par)fat = 'great-great-grandfather'
  • (fat|mot|par)(fat|mot|par)(fat|mot|par)(fat|mot|par)fat = 'great x3 grandfather'
  • (fat|mot|par)(fat|mot|par)(fat|mot|par)(fat|mot|par)(fat|mot|par)fat = 'great x4 grandfather'

This could go on indefinitely! So, we need a way to specify these repeating patterns, and we do it like this:

  • fat = 'father'
  • (fat|mot|par)fat = 'grandfather'
  • (fat|mot|par){2}fat = 'great-grandfather'
  • (fat|mot|par){3}fat = 'great-great-grandfather'
  • (fat|mot|par){4}fat = 'great x3 grandfather'
  • (fat|mot|par){5}fat = 'great x4 grandfather'

This still isn't good enough - as we'd still need separate rules for every possible number of generations. So, we use expressions instead

  • fat = 'father'
  • (fat|mot|par)fat = 'grandfather'
  • (fat|mot|par){2}fat = 'great-grandfather'
  • (fat|mot|par){3}fat = 'great-great-grandfather'
  • (fat|mot|par){4,}fat = CONCAT('great x', @up-2, ' grandfather')

Here, the {4,} means "4 or more repetitions of the previous pattern". The "@up" means the total number of "fat/mat/par" links in the relationship (not the number matched in the repeating pattern).

The expression can use the following values

  • @up (the total number of "fat/mot/par" links in the relationship)
  • @down (the total number of "son/dau/chi" links in the relationship)
  • @sex (the sex of the first person in the path)
  • @marr (for couples - are they married)
  • @div (for couples - are they divorced)
  • @dob1 (the date of birth of the first person)
  • @dob2 (the date of birth of the last person)

For complex relationships, such as cousins, the rules can be long and complicated. The developers can help with the expressions, but we will need a native speaker to carefully explain the relationships!

I invite the translators for each language to add their own rules below. Contact me if you need help with the pattern matching or the expressions.

It is possible to include different rules. For example, canon-law versus civil-law.
