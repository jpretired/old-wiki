Using Sources, Citations, and Repositories

From webtrees
Jump to: navigation, search

It is strongly recommended that a source is given for all the information you enter in your database. You may think it's self evident or trivial, but other users may not, and should conflicting information come up at a later point, you will regret having left it out.

Sources are part of a hierarchy of references, with Citations and Repositories on either side.

  • A Source is where you gathered your information from. It can be anything from a parish register or a photo of a tombstone, to something you remember having heard Aunt Sally say during a family dinner when you were five or a website.
  • A Citation is the actual contents in the Source from which you draw your information. A page number in a book, a line in a parish register, or the location on a web site.
  • A Repository is a collection of sources, such as an Archive. It is the place where the parish register, tombstone or photo is stored. This could also be the website owner.

Entering Source Citations

In every editing window for facts or events, there is an option to "Add a new source citation".

All source citations must come from a specific Source. If you have used the same source before, you can start typing its name in the text field and select it from the drop-down list that appears (provided autocompletion is enabled). You can also click on the "Find Source ID" button next to the text field to search for the source.

When the connection to the source has been made, you can add further information in the remaining fields:

  • Citation details: How to locate the citation in the source. If you're quoting from a book, this will be the page or folio number, but other kinds of sources may require different formats ("To the left of the beech by the church", "http://sourcesite.com/sourcepage.html", etc.)
  • Text: The cited text itself.
  • Multimedia object: You can for example add a scan of a page in a parish register or some other source. This is not only a good way to make your site varied and interesting; it is also a time saver if you have to revisit the source later, and a safeguard should the web page you've taken your scan from be taken down. (NB: if you use remote images, beware of copyright issues; it is your responsibility to make sure you have the rights to reproduce an image).
  • Shared note: Any text you enter in the text field will be stored in the record of the person to which the source is connected, and will not be available from elsewhere. If you want to re-use the text in other records, you should enter it as a Shared note instead.

NB: If you fail to enter a source, the information you enter in other fields will be lost!

Creating new Sources

If this is the first time you use the source, use the "Create a new source" button instead.

Enter the details about the source in the new window that appears. The Title field is required; the others are optional.

  • Abbreviation: This is an optional field to add a short form of the title. Note that in some genealogical programs, the Abbreviation is required.
  • Author and Publication: These fields will be useful if you're citing from a book. For other kinds of sources, they may or may not be of any use.

NB: The Source window is not the place to write quotations from the source.

When you have filled all the fields you want to use, click on "Create a new source".

If you're creating the source from a Citation window, there will be a message telling you to "Paste the following ID into your editing fields to reference the newly created record". This message is a link, and clicking it will do what the message says. If you just close the window, the source ID will not be entered in the citation. The source will not be lost, but you will have to make the connection manually.

Repositories

A source can also belong to a larger collection of sources: an Archive. The procedure to connect a source to a Repository is parallel to the way one adds Citations to Sources.

The Call number is the code by which the source is identified in the Repository.

Reliability

A Source is not the Truth -- it is just a piece of evidence supporting the conclusions you make, for example that someone was born 19 Dec 1769.

Not all sources are equally reliable or credible. In webtrees you can enter your own assessment of the reliability of a source. This is done in the "Quality of Data" field. It is customary to use a numbering scheme for the quality assessment:

0 = Unreliable evidence or estimated data
1 = Questionable reliability of evidence (interviews, census, oral genealogies, or potential for bias, for example, an autobiography)
2 = Secondary evidence, data officially recorded sometime after event
3 = Direct and primary evidence used, or by dominance of the evidence

The "Quality of Data" field is not visible by default in the standard webtrees installation. It is considered to be an advanced feature which must be enabled by the administrator.

Online Sources

These days, many source references are to online resources: scans of church registers, online databases, other genealogical web pages, etc. The GEDCOM specification was not written with this situation in mind, but webtrees provides good tools to handle such sources as well.

If your Citation is from a web page, enter the URL in the "Citation details" field. This is a free-text field, so you can enter additional information as well, such as the page number of a scanned book. It is a good idea to enter the date on which the page was retrieved as well, since web addresses may change.

Any URL you enter in this field, will automatically be converted to a clickable link.

If the page from which you are citing is part of a larger web site the address of which you also want to record, there are different ways to do this.

  • If the web site is part of an even larger site (e.g. scans from a particular book in a "virtual archive"), you can make the main site a Repository and enter the address of the intermediary site as the "Call number". This would be the situation if an archive has published its collections online: The archive's main website would be the Repository; the portal page for the book you're quoting from would be the Source, with the URL given in the Calling number field; and the URL for the page from which you're quoting would be entered in the Citation details field.
  • If the web site is not part of a larger structure, you can still use the Repository--Source--Citation model.
  • If you don't want to designate the web page as a repository, there are other fields where the you URL can be entered. If you navigate to the main page for the source, there is a drop-down list of facts to add. Depending on the settings of the site, a WWW field may be available. Otherwise, you can enter the address in a Note or under Publication details. Currently, any URL in a Note is converted to a clickable link, but this is not the case in the Publication field.
    NB: Using the WWW tag for a source URL will produce GEDCOM code which is not strictly according to the standard. You will have to decide whether this is a problem or not.
