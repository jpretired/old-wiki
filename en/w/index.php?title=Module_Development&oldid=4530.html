General module creation information

Please note, that your code should follow the Coding Standards. This way, if you abandon your module, any other person/developer can fork your module and continue using it. Also make sure, you have PHPDoc-compatible documentation for each element (classes, functions, files, and variables).

Modules for webtrees 1.7.x

Webtrees 1.7 and above uses a new framework which provides a php composer plugin.

There is a github module skeleton at https://github.com/bmhm/webtrees-module-skeleton.

Having your IDE recognize classes

Using composer is highly encouraged. Add this requirement: https://packagist.org/packages/fisharebest/webtrees.

Module for webtrees 1.6.x

Basic information for webtrees 1.6-modules

For webtrees 1.6, you need to follow some basic requirements:

  • The plugin-folder is /modules_v3/.
  • Each module gets a seperate subfolder. It may not contain dashes (-) or special chars, just underscores and alphanumeric chars.
  • The entrypoint for each module is the file module.php.
  • The file needs to implement a class with the name [a-zA-Z0-9_]_WT_Module.

So, a basic module.php has a class which extends WT_Module and overrides some basic methods, like so: 

   class jsonld_WT_Module extends WT_Module implements WT_Module_Tab {
   	
   	/* ****************************
   	 * Module configuration
   	 * ****************************/
   	public function getTitle() {
   		return "JsonLD";
   	}
   	
   	public function getDescription() {
   		return "Adds json-ld-data to persons as described in schema.org/Person";
   	}
   	
   	/** {@inheritdoc} */
   	public function defaultAccessLevel() {
   		return WT_PRIV_PUBLIC;
   	}
   }

This is a fully working module with no functionality, which can be enabled in the admin web interface. To extend functionality, you need to either override modAction($action) or implement an Interface like tab, menu, etc. Please note, that this example does not use namespaces, and also mandatory comments are missing.

Having your IDE recognize classes

In 1.6.x, there is no composer plugin. But you can use the webtrees source folder as a git submodule (an external dependency). Make sure, after pulling webtrees, you switch to tag 1.6.2 (see https://github.com/fisharebest/webtrees/tree/1.6.2). Then include the submodule in your favourite IDE as library/include folder.

Please do not check in the whole webtrees branch, just the submodule reference.

You can also use composer, see https://packagist.org/packages/fisharebest/webtrees.
