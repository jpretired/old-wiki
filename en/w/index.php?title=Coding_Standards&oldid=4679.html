Language

The code and comments must be written in U.S. English.

PHP Coding Standards

The PHP coding standards mostly follow the PSR standards. Comments and exceptions are noted below.

PSR-1 - Basic Coding Standard

  • Local variables, function arguments and member properties should be named consistently (section 4.2). We use "snake_case" for these. For example:
   class FamilyTree {
   
     const MAGIC_NUMBER = 42;
   
     private $some_property;
   
     public function createTree($function_argument) {
       $local_variable = $function_argument + 1;
     }
   }
   
   function some_global_function() {
     static $remember_me;
   }

PSR-2 - Coding Style Guide

Indentation (section 1)
We use TABS for indentation (and only for indentation). We use spaces for alignment.
Opening braces (section 1)
Opening braces for classes and functions go on the previous line, as per other control structures such as if, while, foreach. For example:
   class FishClass {
     public function whatAreFish() {
       return 'best';
     }
   }
Line length (section 2.2)
120 characters is a reasonable limit for most displays - but readability is more important than arbitrary limits. Longer lines are both permitted and encouraged, where they aid readability. Wide-screen displays are commonplace.
When lines are split, the joining operator should be placed on the end of the preceding line. For example:
   $text =
     'The beginning ' .
     'and the end'
