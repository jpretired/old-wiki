Terminology

From webtrees
Jump to: navigation, search


For the benefit of users, translators and developers, we should endeavour to be consistent in our use of language and terminology.

When learning a new piece of software - especially one as complex as webtrees - it is necessary to learn the objects and concepts involved. It can be very confusing if an application refers to the same thing using two different names. For this reason, we should always refer to objects and concepts using the same terms.

Abbreviations

Avoid abbreviations and short codes wherever possible. For example, use "North/South" instead of "N/S". Using full words makes things easier for our users and translators alike.

Prefered terms

  • Create - rather than add a new record
  • Facts and events are different things. Use "facts and events" or "facts or events" when you need to refer to both.
  • Family tree - rather than GEDCOM, although it is OK to refer to GEDCOM files, GEDCOM tags, etc. if that is what you mean.
  • Filename - rather than file name, although constructs such as "file name and size" are better with separate words.
  • Folder - rather than directory. Directory is a technical/implementation term. The majority of other GUI applications use folder.
  • Google Maps™ and Google Street View™ - are trademarks of Google and should be indicated as such
  • Individual - rather than person
  • Media - rather than multimedia
  • Media file - rather than multimedia file
  • Media object - rather then multimedia object/item
  • Preferences - rather than settings, configuration, options
  • Username - rather than user name or user-name
  • Zoom level - rather than zoom factor

When requesting accounts, new users verify their email addresses and administrators approve the account.

Capitalization

Capitals should be used for proper nouns and the start of sentences. Do not capitalize all the initial letters to make a title (e.g. "Edit Media Object").

Button labels

Button labels should be lower-case verbs. For example "continue", "save", "cancel", "close". Note that "OK" is not a verb. Use continue/save as appropriate.

Field labels

Field labels should just describe the values, and not include the act of choosing them. For example, avoid labels such as "Enter the search terms" or "Select a language". Just use "Search terms" or "Language". These "instructional" labels cannot always be translated without knowing the sex of the user.

Punctuation

Use the typographical punctuation characters, rather than the keyboard " and ' characters. They look much better.

We write in US English, so use “double quotation marks” rather than ‘single quotation marks’, « guillemets », bold etc.
