Custom Tags

From webtrees
Jump to: navigation, search

Many users, especially novice users, seek to extend the GEDCOM specification with their own tags. This page does three things:

  1. It tries to dissuade you from doing it.
  2. It gives you alternatives.
  3. It explains how to do it. This is not because we support, encourage or endorse the technique. It simply saves us having to explain it again and again. There are probably other ways to do it, this is just one.

Reasons to avoid custom tags

There are many reasons to avoid custom tags.

  1. Other applications will ignore (or even delete) them.
  2. No meaning or significance can be assigned to them.
  3. Other genealogists will not know what they mean.
  4. The GEDCOM specification advises against it.
  5. It makes it difficult to migrate your data to other applications.
  6. The webtrees developers think it is a bad idea. We don't do it ourselves, and we don't support it.

Alternatives to custom tags

Use the "TYPE" qualification with an existing tag

Professional actors and entertainers frequently use a stage name. Instead of creating a special tag for "Stage name", you could use 

1 NAME John /Wayne/
2 TYPE stage name

Use an existing custom tag

A number of custom tags have been created by various desktop applications, and webtrees recognises many of these. Instead of creating a tag to indicate that someone was blind, you could use an existing "medical" tag. 

1 _MDCL Blind from birth

Although webtrees can translate the label for these tags, only a very few of them are given any meaning. For example, the _NMR (not married) and _MARNM (married name) tags.

Note: support for these tags exists only for historical reasons. If we were starting again, we probably wouldn't do it. Instead, we'd map them all onto EVEN, FACT or NOTE tags.

Use a NOTE

You can record almost anything in a note. Instead of creating a tag to record that someone was illiterate, you could use a note: 

1 NOTE Illiterate - signed all documents with an X

But if you really must ...

To add a tag _FOOBAR

Add the tag to the user interface

From the family tree configuration page, select the "Edit" tab, and add _FOOBAR to the appropriate individual/family/source/etc. record

Add the tag to the internal list

Edit the file app/GedcomTag.php and:

  1. add _FOOBAR to the definition of $ALL_TAGS
  2. add the English label for the tag in the getLabel() function, e.g. 'Foo Bar'. You may need to create separate labels for use with males and females. Copy the code for one of the existing tags.

Add translations into other languages

Create a translation for 'Foo Bar' into each of your supported languages, using the instructions in the Custom Translation page. Untranslated languages will display the English text.
