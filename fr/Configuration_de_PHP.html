Configuration de PHP

De webtrees
Aller à : navigation, rechercher

Quelques exigences de webtrees (voir Configuration requise) amènent aux informations suivantes à propos de PHP :

  • PHP 5.2.3 ou version ultérieure. Notez que de nombreux hébergeurs web offrent à la fois PHP4 et PHP5 souvent avec PHP4 par défaut. Si c’est le cas, vous serez en mesure de basculer entre les deux en utilisant un panneau de commande ou un fichier de configuration. Consultez la documentation du support de votre hébergeur pour plus de détails.
    • PHP doit être configuré avec la bibliothèque de PHP/PDO pour MySQL. Il s’agit d’une option de configuration du serveur. Il est activé par défaut sur la plupart des hôtes. Voir http://php.net/pdo
    • PHP doit être configuré pour autoriser les ressources suffisantes du serveur (mémoire et temps d’exécution) pour la taille de votre système. Les exigences de base sont :
      • Les petits gedcoms (500 personnes) : 16-32 Mo, 10-20 secondes
      • les gedcoms moyen (5000 personnes) : 32-64, 20-40 secondes
      • les grands gedcom (50000 personnes) : 64-128, 40-80 secondes

Le processus visant à définir ces éléments de configuration et d’autres peuvent varier en fonction de votre hébergeur. La configuration est définie dans le fichier php.ini par le système, ou est modifiée par l’utilisateur ou par l’hébergeur.

Malheureusement, certains hôtes ne permettent pas cette option. Une alternative est offerte par certains hébergeurs en créant un fichier .htaccess contenant les paramètres de configuration. Il est ensuite stocké à la racine de l’installation de webtrees (l’emplacement exact peut varier selon le serveur).

(REMARQUE IMPORTANTE - ceux-ci sont souvent spécifiques à certains hébergeurs et peuvent être similaires, mais non identiques. Toujours vérifier l'exactitude de chaque détail pour votre hébergeur...)

Activation de php5

Concentric.com indique : "Nous soutenons actuellement la version 5.2.3, 4.4.7 et 3 de PHP La valeur par défaut est php4. La seule façon d’utiliser la version 5 sera une directive dans htaccess. La directive htaccess peut aussi convenir pour utiliser php3. Pour ajouter php5 CGIMapExtension/usr/local/bin/php5.php

date_timezone

Il s'agit du fuseau horaire permettant d'afficher l'heure correcte.

Lunarpages indique : "Créer ou modifier htaccess dans le répertoire racine du site avec une ligne SetEnv TZ 'position'. Cette "position" peut être trouvée sur http://www.php.net/manual/fr/timezones.php. Pour la France métropolitaine, c'est "Europe/Paris"

Un exemple complet pourrait être : SetEnv TZ Europe/Paris. Ce fichier n'adaptera l'heure que pour le site à la racine duquel il se trouve.

En local, le problème ne devrait pas se poser parce qu'il y a de fortes chances que le fichier de configuration de php soit configuré pour afficher votre heure correctement. Si ce n'était pas le cas, éditez le fichier php.ini (sous EasyPhp, il se trouve dans le dossier conf_files de easyPhp) et recherchez la ligne débutant par "date.timezone =". Enfin, remplacez la fin de la ligne par "Europe/Paris" pour obtenir "date.timezone = Europe/Paris". Ce fichier (php.ini) existe aussi sur un site distant mais, si vous ne gérez pas votre serveur, il n'y a guère de chance que votre hébergeur vous laisse y accéder. ATTENTION: restez très prudent dans les modifications de ce fichier de configuration et n'agissez qu'en connaissance de conséquence (ne modifiez aucune autre ligne si vous n'êtes pas sûr de vous).
