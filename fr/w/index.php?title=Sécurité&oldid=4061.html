La sécurité dans webtrees vise à s'assurer que votre site est protégé contre les intrusions non souhaitées, le piratage ou l'accès aux données et aux fichiers de configuration. Les développeurs de webtrees considèrent la sécurité comme une partie extrêmement importante de son développement et font tout pour que vos données soient en sécurité.

La zone la plus à risque d'intrusion est sans doute le répertoire /data qui contient votre fichier config.ini.php et divers fichiers temporaires. Si vous souhaitez savoir si un risque existe, vous pouvez faire un test très simple : faites une recherche sur config.ini.php en tapant http://url_de_votre_site/data/config.ini.php dans votre navigateur.

Normalement, vous devriez obtenir le message "accès refusé" (erreur 403...) sans doute écrit en anglais comme ceci :

Forbidden

You don't have permission to access /data/xxxx.ged on this server.

Cela signifie que la protection intégrée dans webtrees fonctionne bien. Aucune action supplémentaire n'est nécessaire.

Dans le cas, assez improbable, où le fichier serait accessible, vous obtiendriez simplement un point-virgule. Et donc, si la protection semblait ne pas fonctionner sur votre site, vous devriez prendre des mesures supplémentaires. Si vous obtenez une erreur "404..." (page inexistante), c'est, sans doute, dû à une erreur de saisie dans l'URL tapée. Vous devriez vérifier ce que vous avez tapé.

Si votre serveur exécute PHP en mode CGI, changez les permissions du répertoire /data à 700 à la place de 777. Ceci va bloquer l'accès au processus httpd, tout en permettant l'accès aux scripts PHP.

Cela fonctionne pour, sans doute, 99 % des utilisateurs. Seul, le 1 % restant devraient envisager la solution beaucoup plus complexe de déplacement du répertoire /data hors de cet espace web trop facilement accessible (note : chez de nombreux hébergeurs, en mode mutualisé, ce n'est généralement pas une option possible).

Si vous trouvez que c'est nécessaire, suivez cet exemple du processus requis :

Si votre répertoire de base est quelque chose comme /home/nom/ et le répertoire racine de votre site web est /home/nom/public_html, et si vous avez installé webtrees dans sous-répertoire /public_html/webtrees/, alors vous devez créer un nouveau répertoire data dans le répertoire de base (/home/nom/) au même niveau que votre répertoire public_html, tel que /home/nom/private/data, pour y déposer votre GEDCOM (arbre généalogique).

Ensuite, dans webtrees, il faut changer le répertoire des fichiers de données dans Administration > Administration du site > Configuration du site en remplaçant le répertoire par défaut data/ par le nouvel emplacement /home/nom/private/data/

Vous trouverez deux répertoires data:

  • [chemin webtrees]/data - qui contient seulement config.ini.php
  • /home/nom/private/data - qui contient tout le reste

Hypertext Transfer Protocol Secure (HTTPS)

webtrees supporte l'accès https. Si votre site est configuré avec l'option https obligatoire ou facultative, webtrees fonctionnera correctement dans chaque mode.

Si votre site est configuré pour l'option https, webtrees peut être configuré pour utiliser la connexion https. Pour utiliser la connexion https, réglez le paramètre URL de connexion dans webtrees (Administration -> Administration du site -> Configuration du site) selon votre URL de connexion https, qui ressemble souvent à https://votrenomdedomaine.fr/admin.php. Attention : une mauvaise configuration peut désactiver votre lien de connexion. Si cela se produit, accédez à la connexion en tapant l'URL correcte directement dans la barre d'adresse de votre navigateur.
