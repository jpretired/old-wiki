Plusieurs utilisateurs, en particulier les débutants, cherchent à accroître la norme GEDCOM avec leurs propres balises. Cette page fait trois choses :

  1. Elle essaie de vous dissuader de le faire
  2. Elle vous donne des alternatives
  3. Elle vous explique comment faire. Ce n'est pas parce que nous appuyons, encourageons ou endossons cette technique. C'est simplement pour éviter d'avoir à répéter les explications indéfiniment. Il y a probablement d'autres façons de le faire, celle-ci en est juste une.

Raisons pour éviter les balises personnalisées

Il y a plusieurs raisons d'éviter les balises personnalisées.

  1. Les autres applications les ignoreront (ou même les supprimeront)
  2. Aucun sens ou signification ne peut leur être attribué
  3. Les autres généalogistes ne connaîtront pas leur signification
  4. La norme GEDCOM conseille de ne pas le faire
  5. La migration de vos données vers d'autres applications peut être difficile
  6. Les développeurs de webtrees pensent que c'est une mauvaise idée. Nous ne le faisons pas nous-mêmes et nous n'offrons pas d'aide pour ces balises personnelles.

Alternatives aux balises personnalisées

Utilisation du champ "TYPE" d'une balise existante

Les acteurs professionnels utilisent fréquemment un "nom d'artiste" ou "nom de scène". Plutôt que de créer une balise personnelle, vous pouvez utiliser 

1 NAME John /Wayne/
2 TYPE nom d'artiste

Utilisation d'une balise usuelle déjà existante

Un certain nombre de balises usuelles a été créé par différentes applications de bureau et webtrees reconnaît plusieurs d'entre elles. Plutôt que de créer une balise pour indiquer qu'un individu était aveugle, vous pouvez utiliser la balise existante "médical". 

1 _MDCL Aveugle de naissance

Même si webtrees peut traduire l'étiquette de ces balises, un sens n'est attribué qu'à quelques-unes d'entre elles. Par exemple, les balises _NMR (non marié) et _MARNM (nom après mariage).

Note: l'aide pour ces balises existe seulement pour des raisons historiques. Si nous avions à recommencer, nous ne le ferions probablement pas. À la place, nous les aurions toutes regroupées sous les balises EVEN, FACT ou NOTE.

Utilisation de la balise NOTE

Vous pouvez enregistrer à peu près n'importe quoi dans une note. Plutôt que de créer une balise pour enregistrer l'information qu'un individu était illettré, vous pouvez utiliser une note : 

1 NOTE Illettré - signe tous les documents par un X

Mais si vous devez vraiment...

Ces instructions s'appliquent à la version webtrees 1.1.2. Elles ne s'appliquent pas aux versions antérieures et peuvent ne pas s'appliquer aux versions ultérieures.

Pour ajouter une balise _FOOBAR

Ajout de la balise à l'interface utilisateur

Sur la page de configuration de l'arbre généalogique, cliquer sur l'onglet Édition et ajouter la balise _FOOBAR aux enregistrements appropriés d'un individu/d'une famille/d'une source/etc.

Ajout de la balise à la liste interne

Modifier le fichier library/WT/Gedcom/Tag.php et :

  1. ajouter _FOOBAR à la définition de $ALL_TAGS
  2. ajouter l'étiquette English de la balise dans la fonction getLabel(), e.g. 'Foo Bar'. Vous aurez à créer des étiquettes séparées pour l'utiliser avec hommes et femmes. Copier le code d'une des balises existantes.

Rappelez-vous que vous devrez refaire ces modifications chaque fois qu'il y aura une mise à jour de version de webtrees.

Ajout de traductions en d'autres langues

Créer une traduction de 'Foo Bar' dans chacune des langues utilisées par votre site webtrees, référez-vous aux instructions de la page Traductions personnalisées. Le texte dans les langues non-traduites affichera le texte anglais.
