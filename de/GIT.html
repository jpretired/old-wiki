Was ist GIT?

GIT ist ein Versions-Kontrollsystem, das von webtrees verwendet wird, um seinen Quellcode zu verwalten.

Das webtrees Entwicklungsteam benutzt viele der GIT-Funktionen wie verzweigen und zusammenführen. Nicht-Entwickler werden in der Regel nur eine Funktion verwenden – nämlich die Fähigkeit, die letzte Code-Version abzurufen.

Das GIT-Archiv befindet sich unter https://github.com/fisharebest/webtrees.

Falls Sie GIT nicht verstehen, aber mit SUBVERSION vertraut sind, bietet github.com eine SVN-Schnittstelle.

Warum Sie Entwicklungs-Code von GIT NICHT VERWENDEN SOLLTEN

  • Sie sind voller Fehler. Ernsthaft! Wenn Sie Code haben wollen, der unvollständig, ungetestet oder schlicht kaputt ist, dann sind Sie hier richtig.
  • Sie werden nicht unterstützt. Das Projekt bietet Unterstützung für die offiziellen Versionen. Wenn wir Zeit haben, werden wir versuchen, zu helfen, aber unsere Priorität liegt auf der Unterstützung der letzten offiziellen Version.
  • Sie können Design-Änderungen enthalten, die nicht einfach rückgängig gemacht werden können.

Unterschiede zwischen Entwicklungs-Code und offiziellen Versionen

Der Entwicklungs-Code enthält nur die Übersetzung für U.S. Englisch. Wenn Sie andere Sprachen wollen, müssen Sie die .PO Dateien erzeugen.

Die statischen Bestandsdateien (.CSS und .JS) sind für die Versionen komprimiert. Der Entwicklungs-Code enthält nicht-komprimierte Dateien. Das hat einen kleinen Einfluss auf die Leistung, wenn Sie Entwicklungs-Code auf einem Produktionsserver laufen lassen.

Warum Sie Entwicklungs-Code VIELLEICHT VERWENDEN MÖCHTEN

  • Sie sind ein Entwickler und möchten Patch-Dateien erstellen.
  • Sie sind ein Entwickler und möchten Änderungen in Ihrer Arbeitskopie mit Änderungen aus dem Stamm verschmolzen haben.
  • Sie sind ein Tester und möchten an Fehlern, die behoben, aber noch nicht veröffentlicht wurden, testen.

Sichern, Sichern und Sichern

Bitte vertrauen Sie Ihre wertvollen Daten nicht einer Entwickler-Version an. Sie sollten eine vollständige Sicherung der Datenbank vor der Benutzung einer Entwickler-Version und regelmäßig danach erstellen.
