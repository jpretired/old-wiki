webtrees ist in Englisch geschrieben und benutzt Englisch intern. Das GEDCOM Datei Format benutzt ebenfalls englische Schlüsselwörter und Kennzeichen. Aber weil Menschen eine Vielfalt von Sprachen sprechen, benutzen wir Übersetzungsdateien; um Text in jede Sprache umzuwandeln. Die Übersetzungsdateien werden im Ordner language/ gespeichert.

Beachten Sie, dass wir auch Übersetzungen in regionale Varianten von Englisch haben. Deshalb gibt es separate Übersetzungen für "British English" und "American English".

Sie können diese Übersetzungen durch Hinzufügen weiterer Übersetzungsdateien anpassen und diese Seite erklärt, wie das gemacht wird. Aber bevor wir beginnen, sollten Sie überlegen, warum Sie diese Änderung machen wollen. Wenn es dazu dient, eine Übersetzung zu verbessern oder zu korrigieren, dann sollte es vorzugsweise in die offizielle Übersetzungsdatei aufgenommen werden, so dass alle davon profitieren können. Hierfür müssen Sie nur ein Benzutzerkonto erstellen bei launchpad.net und hier einen Vorschlag machen. Sie müssen dafür nicht um Erlaubnis fragen oder einem Team beitreten. Innerhalb einiger Tage sollte jemand Ihren Vorschlag überprüfen und danach wird er in die nächste Version aufgenommen.

(Anmerkung des Übersetzers: Alle englischen Ausdrücke von webtrees (neue und alte) können in Launchpad gefunden werden. Sie müssen nicht hinzugefügt werden, weil sie standardmäßig aufgenommen werden und aus dem Original PHP Code stammen. Mit dem Suchfeld kann danach gesucht werden. Nur die Übersetzungen in andere Sprachen müssen als 'neue Vorschläge' entweder erstmals oder später als verbesserte oder passendere Version hinzugefügt werden.)

Wo werden lokale Sprachdateien gespeichert

Sie sollten einen Unterordner "language" im webtrees Ordner data/ erstellen, z.B. data/language/. Falls Sie den data/ Ordner außerhalb des Root-Ordners Ihres Webservers angelegt haben, dann sollte er in Ihrem neuen, privaten data/ Ordner liegen.

Dateinamen und Formate

Die Dateinamen bestehen aus dem Sprachkürzel und einer Erweiterung zur Kennzeichnung des Formats.

Beispiele für Sprachkürzel sind:

  • it (Italienisch)
  • fr (Französisch)
  • de (Deutsch)
  • [email protected] (Serbisch in lateinischer Schrift)
  • en_AU (Australisches Englisch)

Drei Formate können verwendet werden und diese werden im folgenden beschrieben. Die Übersetzungen sind immer von Englisch in die Zielsprache.

Beachten Sie, dass bei Übersetzungen immer zwischen Groß- und Kleinschreibung unterschieden wird, so dass "Hallo" anders übersetzt wird als "hallo".

CSV Format

Das einfachste Format ist CSV. Obwohl CSV für "comma separated values" steht, verwenden wir tatsächlich Semikolons anstelle von Kommas, und jeder Text muss in Anführungszeichen eingeschlossen werden. Hier ist ein Beispiel einer CSV Datei für Britisches Englisch, die als data/language/en_GB.csv benannt wäre. Um Anführungszeichen im Text zu verwenden, muss ein Backslash "\" vorangestellt werden. 

"yes";"certainly"
"no";"perhaps not"

Dies würde verursachen, das Wort "yes" innerhalb des ganzen Programms durch "certainly" zu ersetzen.

Die Vorteile dieses Formats sind:

  • Einfachheit. Sie können diese Dateien mit einem einfachen Texteditor oder Tabellenkalkulation erstellen.

Die Nachteile dieses Formats sind:

  • Sie können nicht für komplexe Übersetzungen wie Pluralformen (verschiedene Sprachen haben verschiedene Wege Pluralformen auszudrücken, abhängig von der Anzahl der Objekte) oder mehrere Kontexte (wo das gleiche Wort viele Übersetzungen hat, je nach Kontext) verwendet werden.
  • Je nach Server-Konfiguration kann es für einen Besucher auf Ihrer Website möglich sein, den Inhalt der Datei anzuzeigen und Sie könnten vertrauliche Informationen darin aufgenommen haben.

PHP Format

Das zweite Format verwendet ein einfaches PHP Skript, um eine Matrix von Übersetzungen zu erstellen. Genauso wie die Übersetzungen enthält die Datei eine einfache Überprüfung, um Besucher Ihrer Seite davon abzuhalten, auf die Daten zuzugreifen. Hier ist ein Beispiel einer PHP Datei für Amerikanisches Englisch, die als data/language/en_US.php benannt wäre. Um Anführungszeichen im Text zu verwenden, muss ein Backslash "\" vorangestellt werden. 

<?php
 if (!defined('WT_WEBTREES')) {
  header('HTTP/1.0 403 Forbidden');
  exit;
 }

 return array(
  'yes'=>'for sure!',
  'no'=>'no way!'
 );

Es ist wichtig, dass es keine Leerzeilen oder Leerzeichen vor dem anfänglichen <?php gibt. Das abschließende ?> ist optional - also fügen wir es nicht ein.

Die Vorteile dieses Formats sind:

  • Relativ einfach. Sie können diese Dateien mit einem einfachen Texteditor erstellen.

Die Nachteile dieses Formats sind:

  • Sie können nicht für komplexe Übersetzungen wie Pluralformen (verschiedene Sprachen haben verschiedene Wege Pluralformen auszudrücken, abhängig von der Anzahl der Objekte) oder mehrere Kontexte (wo das gleiche Wort viele Übersetzungen hat, je nach Kontext) verwendet werden.

Gettext Format

Das dritte Format wird gettext genannt. Gettext ist eine open-source, Industrie-Standard Bibliothek für Übersetzungen. Jede Sprache benötigt zwei Dateien. Es gibt eine PO Datei, die menschenlesbaren Text enthält. Obwohl Sie diese Dateien mit Text Editoren bearbeiten *können*, ist es allgemein üblich, einen dafür vorgesehenen "PO Editor" zu verwenden. POEdit ist ein geläufiges Werkzeug dafür, obwohl es eine Reihe von Einschränkungen hat und wir den Better PO Editor empfehlen, der von einem der webtrees Entwickler Michele Locati geschrieben wurde. Die PO Datei muss in eine MO Datei umgewandelt werden, die eine maschinenlesbare, komprimierte Version derselben Datei ist. Möglicherweise müssen Sie die binäre MO Datei aus der webtrees Version nehmen und sie zurück in eine PO Datei umwandeln, um sie mit dem Better PO Editor zu bearbeiten. Nur der POEdit hat das ausführbare msgunfmt, um diese Rück-Übertragung zu leisten. Daher benötigen Sie wohl beide Werkzeuge.

Hier ist ein Beispiel einer PO Datei für Amerikanisches Englisch, das bestimmte Jüdische Begriffe in Aschkenase Schreibweisen umwandelt. Die Datei würde mit en_US.po bezeichnet und Sie müssten sie in eine .mo Datei umwandeln, die als data/language/en_US.mo benannt wäre. Beachten Sie die häufigen kontextabhängigen Übersetzungen, wo es verschiedene Übersetzungen für den selben englischen Text geben kann. 

msgid ""
msgstr ""
"Project-Id-Version: webtrees\n"
"MIME-Version: 1.0\n"
"Content-Type: text/plain; charset=UTF-8\n"
"Content-Transfer-Encoding: 8bit\n"
"Plural-Forms: nplurals=2; plural=n != 1;\n"
"X-Poedit-Language: en\n"
"X-Poedit-Country: US\n"
"X-Poedit-SourceCharset: utf-8\n"

msgctxt "NOMINATIVE"
msgid "Heshvan"
msgstr "Cheshvan"

msgctxt "NOMINATIVE"
msgid "Tevet"
msgstr "Teves"

msgctxt "GENITIVE"
msgid "Heshvan"
msgstr "Cheshvan"

msgctxt "GENITIVE"
msgid "Tevet"
msgstr "Teves"

msgctxt "LOCATIVE"
msgid "Heshvan"
msgstr "Cheshvan"

msgctxt "LOCATIVE"
msgid "Tevet"
msgstr "Teves"

msgctxt "INSTRUMENTAL"
msgid "Heshvan"
msgstr "Cheshvan"

msgctxt "INSTRUMENTAL"
msgid "Tevet"
msgstr "Teves"

msgid "Heshvan"
msgstr "Cheshvan"

msgid "Tevet"
msgstr "Teves"

msgid "Bat mitzvah"
msgstr "Bas mitzvah"

msgid "Date of bat mitzvah"
msgstr "Date of bas mitzva"

msgid "Place of bat mitzvah"
msgstr "Place of bas mitzvah"

msgid "Source for bat mitzvah"
msgstr "Source for bas mitzvah"

Die Vorteile dieses Formats sind:

  • Es kann für alle Übersetzungen einschließlich der komplexen Pluralformen und Kontexte verwendet werden. Es ist das Format, das von webtrees für seine eigenen Übersetzungen verwendet wird.
  • Für Dateien, die eine große Anzahl von Übersetzungen enthalten, ist es schneller und verbraucht weniger Speicher als die anderen Optionen, da es nicht alle Übersetzungen in den Speicher laden muss.

Die Nachteile dieses Formats sind:

  • Es erfordert zusätzliche Software zur Erstellung der Dateien.

Zeichencodierung

Dateien sollten in UTF8 Codierung gespeichert werden, statt ISO-8859, latin, etc.. Sie sollten mit einigen Text Eeditoren besonders vorsichtig sein, weil sie oft ein unsichtbares Zeichen an den Dateianfang stellen (genannt Byte-Order-Mark). Falls Sie nach der Installation Ihrer Datei eine Fehlermeldung in der Form "Headers already sent" erhalten, ist sehr wahrscheinlich ein BOM Zeichen enthalten.

Allgemeine Bemerkungen

  • In welche Sprache Sie auch übersetzen wollen, Ihre benutzerdefinierte Sprachdatei muss immer ausgehend vom Text im Programm-Kern übersetzen (WT_I18N::translate('xxxxx'), nicht von irgendeiner der Standard Sprachdateien. Wenn Sie diese Texte auf dem Bildschirm sehen wollen, stellen Sie sicher, dass Sie "en-US" anzeigen, statt eine der anderen Sprachen.
  • Die Übersetzungs-Bibliothek macht intensiven Gebrauch von Zwischenspeicher. Deshalb müssen Sie nach Hinzufügen Ihrer benutzerdefinierten Übersetzung entweder die Zwischenspeicher Dateien löschen (aus Ihrem System Temp Ordner) oder mindestens zwei Stunden warten. Der genaue Speicherort Ihrer Zwischenspeicher und die Zeit zum automatischen Löschen können von Ihren Einstellungen abhängen.
  • Beachten Sie, dass Sie die Änderungen an dieser Datei jedes Mal vornehmen müssen, wenn Sie auf eine neuere Version von webtrees updaten. Das Online Update Werkzeug (verfügbar ab Version 1.5.1) erstellt eine Datensicherung dieser Datei als teil des Updates, so dass Ihre Änderungen nicht verloren gehen.
