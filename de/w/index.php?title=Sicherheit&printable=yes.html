Sicherheit

Aus webtrees
Wechseln zu: Navigation, Suche

Sicherheit in webtrees bedeutet, zu gewährleisten, dass Ihre Website vor unerwünschtem Eindringen, Hacking oder Zugriff auf Daten- und Konfigurations-Dateien sicher ist. Die Entwickler von webtrees betrachten Sicherheit als äußerst wichtigen Teil ihrer Entwicklung und haben jede Anstrengung unternommen, sicherzustellen, dass Ihre Daten geschützt sind.

Der am stärksten von Eindringen gefährdete Bereich ist sicher der /data Ordner, der Ihre config.ini.php Datei und verschiedene temporäre Dateien enthält. Falls Sie besorgt sind, dass es ein Risiko geben könnte, gibt es einen sehr einfachen Test für Sie: Versuchen Sie durch Eingabe von http://adresse_ihrer_website/data/config.ini.php in Ihrem Web-Browser auf die config.ini.php zuzugreifen.

Das wahrscheinlichste Ergebnis ist eine Meldung "Zugriff verweigert" wie diese:

Forbidden

You don't have permission to access /adresse_ihrer_website/data/config.ini.php on this server.

Dies deutet darauf hin, dass der in webtrees eingebaute Schutz funktioniert und keine weitere Maßnahme erforderlich ist.


In dem unwahrscheinlichen Fall, dass Sie auf die Datei zugreifen können (Sie sehen nur ein Semikolon), funktioniert der Schutz auf Ihrer Website nicht und Sie sollten weitere Maßnahmen ergreifen.

Wenn Ihr Server PHP im CGI-Modus betreibt, dann ändern Sie die Verzeichnisrechte für den /data Ordner auf 700 anstelle 777. Dies wird den Zugang zum httpd-Prozess unterbinden, während auf PHP-Skripte immer noch zugegriffen werden kann.

Dies wird vielleicht für 99% aller Benutzer funktionieren. Nur die restlichen 1% sollten die komplexeste Lösung in Erwägung ziehen und den /data / Ordner aus dem zugänglichen Webspace entfernen. (Hinweis: In vielen geteilten Hosting-Umgebungen ist dies jedenfalls keine Option.)


Wenn Sie es für erforderlich halten, finden sie nachfolgend ein Beispiel für die Vorgehensweise:

Wenn Ihr Home-Ordner so etwas ist wie /home/username und der Root-Ordner für Ihre Website /home/username/public_html und Sie haben webtrees im public_html/webtrees Ordner installiert, dann würden Sie einen neuen data-Ordner in Ihrem Home-Ordner auf gleichem Niveau wie Ihr public_html Verzeichnis erzeugen, so wie /home/username/private/data und Ihre GEDCOM-Datei (Stammbaum) dort platzieren.

Dann ändern Sie den Data folder auf der Seite Verwaltung – Site-Einstellungen von der Standard-Einstellung data/ auf den neuen Ort /home/username/private/data.

Sie haben dann zwei Daten-Verzeichnisse:

  • [Pfad zu webtrees]/data - muss nur config.ini.php enthalten
  • /home/username/private/data - enthält alles andere


Hypertext Transfer Protocol Secure (HTTPS)

webtrees unterstützt https-Zugriff. Wenn Ihre Website verbindlich oder optional mit https Unterstützung konfiguriert ist, wird webtrees in beiden Modi korrekt funktionieren.

Wenn Ihre Website optional mit https-Unterstützung konfiguriert ist, kann webtrees so eingestellt werden, dass es bei der Anmeldung auf https umschaltet. Um https bei der Anmeldung zu aktivieren, setzen Sie die Einstellung für Login-URL auf der Seite Verwaltung – Site-Einstellungen auf Ihre https Anmelde-Adresse, die oft die Form https://example.com/admin.php hat (setzen Sie Ihre Domain für example.com ein).

Wenn Sie die Adresse der Webseite auf der Verwaltungsseite Site-Einstellungen auf eine http-Adresse gesetzt haben, hat das Vorrang und Ihre Anmeldeseite wird auf http geleitet. Die Adresse der Webseite muss wahrscheinlich leer sein, um die Anmelde-Adresse zu verwenden.

Achtung: Eine Fehlkonfiguration kann Ihre Anmelde-Verbindung deaktivieren. Falls das passiert, können Sie auf die Anmeldung durch Eingabe der korrekten Adresse direkt in die Adresszeile Ihres Browsers zugreifen.
