PHP Programmier-Vorgaben

Die PHP-Programmier-Vorgaben folgen meistens den PSR-Standards.

PSR-0 - Vorgaben für Automatisches Laden

  • Wir haben immer noch eine große Anzahl von globalen Funktionen (in /includes/functions/...). Diese müssen in Klassen verschoben werden, so dass sie automatisch geladen werden können. Diese Arbeit ist im Gange.

PSR-1 - Grundlegende Programmier-Vorgaben

  • In der Version 1.5.0 sind wir von PHP 5.2 auf PHP 5.3 übergegangen. Deshalb sollten wir jetzt formale Namensräume (Abschnitt 3) verwenden. Wegen der Auswirkungen auf Drittanbieter-Module muss diese Bearbeitung wahrscheinlich mit einem Release-Wechsel (z.B. 1.6.0 oder 2.0.0) kombiniert werden.
  • Lokale Variablen, Funktionsargumente und Element-Eigenschaften sollten durchgängig (Abschnitt 4.2) benannt werden. Wir verwenden dafür "snake_case". Zum Beispiel:
   class FamilyTree {
   
     const MAGIC_NUMBER = 42;
   
     private $some_property;
   
     public function createTree($function_argument) {
       $local_variable = $function_argument + 1;
     }
   }
   
   function some_global_function() {
     static $remember_me;
   }

PSR-2 - Programmierstil-Anleitung

Einzug (Abschnitt 1)
Wir verwenden TABS für Einzüge (und nur für Einzüge). Wir nutzen Leerzeichen für die Ausrichtung.
Öffnende Klammern (Abschnitt 1)
Öffnende Klammern für Klassen und Funktionen stehen in der vorhergehenden Zeile nach anderen Kontrollstrukturen wie if, while, foreach. Zum Beispiel:
   class FishClass {
     public function whatAreFish() {
       return 'best';
     }
   }
Zeilenlänge (Abschnitt 2.2)
120 Zeichen ist eine vernünftige Grenze für die meisten Displays - aber die Lesbarkeit ist wichtiger als willkürliche Grenzen. Längere Zeilen sind sowohl erlaubt als auch erwünscht, wenn es der Lesbarkeit dient. Breitbild-Displays sind an der Tagesordnung.
Wenn Zeilen umgebrochen werden, sollte das Verbindungs-Zeichen am Ende der vorhergehenden Zeile platziert werden. Zum Beispiel:
   $text =
     'The beginning ' .
     'and the end'
