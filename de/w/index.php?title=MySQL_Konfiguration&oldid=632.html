MySQL hat eine große Anzahl von Konfigurationsoptionen. Diese einzustellen, kann einen riesigen Unterschied für die Leistung Ihrer Datenbank machen. Leider gibt es nicht den einen "besten" Satz an Einstellungen. Die optimale Konfiguration hängt von der MySQL-Version, den Server-Ressourcen, anderen Server-Prozessen, etc. ab. Sie ist auch abhängig vom Server-Einsatz, wie Entwicklung, Test oder Live-Betrieb.

Die Standardkonfiguration, die von vielen Installationen bereitgestellt wird, ist sehr schlecht. Insbesondere basiert die Standardkonfiguration mit XAMPP auf einem Server mit 64 MB RAM und einer sehr alten Version von MySQL.

Die Einstellungen, die hier vorgeschlagen werden, sind ein Kompromiss zwischen Leistung, Features, Daten-Integrität und Protokollierung.

Konfigurations-Datei

Die Konfigurationseinstellungen werden in einer Textdatei gespeichert. Diese Datei wird in der Regel unter Windows als my.ini und unter Linux und MacOS-X als my.cnf bezeichnet. MySQL liest diese Datei, wenn es gestartet wird. Wenn Sie die Einstellungen ändern, müssen Sie MySQL neu starten, damit alle Änderungen wirksam werden. Erstellen Sie eine Sicherungskopie der ursprünglichen Datei bevor Sie beginnen.

Wenn irgendwelche Fehler in my.ini enthalten sind, wird MySQL sich weigern zu starten und eine Nachricht in die Fehler-Datei (in der Regel) mysql.err ausgeben, um zu erklären warum.

my.ini Struktur

my.ini Dateien werden in eine Anzahl von Abschnitten unterteilt. Wir sind nur mit dem Abschnitt in Bezug auf den MySQL-Daemon-Prozess mysqld beschäftigt. Dies kann der einzige Abschnitt sein. Zeilen, die mit # beginnen, sind Kommentare, Leerzeilen werden ignoriert und Konfiguration-Zeilen haben das Format Einstellung=Wert. Dieses Beispiel zeigt jede Art von Zeile. 

# Konfigurations-Einstellungen für den MySQL Daemon Prozess

[mysqld]
port=3306

Bevor Sie beginnen

Sie sollten eine Kopie der vorhandenen my.ini-Datei machen, bevor Sie irgendwelche Einstellungen ändern. Es ist wahrscheinlich eine gute Idee, Ihren gesamten MySQL-Daten-Ordner zu sichern.

Sie sollten die Version Ihres MySQL servers herausfinden (sie kann sich unterscheiden von der Version Ihres MySQL clients). Einige der Konfigurationseinstellungen hängen von Ihrer MySQL-Version ab. Dies wird auf der Homepage von phpMyAdmin angezeigt.

HINWEIS: Während die meisten Konfigurationsmöglichkeiten durch einfaches Bearbeiten in my.ini geändert werden können, gibt es eine Dateigrößen-Einstellung (innodb-log-file-size), die einige zugehörige Änderungen im Datei-System erfordern. Ändern Sie diese Einstellung nicht, ohne die zugehörigen Kommentare zu lesen und zu befolgen.

Einstellungs-Vorschläge

Alle diese Einstellungen gehören in den [mysqld]-Abschnitt der my.ini Konfigurationsdatei.

Verbindung und Konfiguration

MySQL benötigt die aktuelle Uhrzeit, um Zeitstempel zu generieren, etc.. SYSTEM bedeutet, den Wert aus dem Betriebssystem zu übernehmen. 

default-time-zone=SYSTEM

Dateien und Ordner können mit absolutem oder relativem Pfad angegeben werden. Wenn Sie mehr als ein Laufwerk haben, legen Sie tmpdir auf ein anderes Laufwerk als datadir. Diese Werte zeigen eine typische XAMPP-Installation. 

basedir=c:/xampp/mysql/
datadir=data/
tmpdir=c:/windows/temp/

# Windows
pid-file=mysql.pid

# Linux
#pid-file=/var/run/mysqld/mysqld.pid

Dies ist die Größe der größten Abfrage, die der Server akzeptiert. Aufgrund eines Fehlers in einigen Versionen von MySQL, beschränkt sie auch die Größe der BLOB-Spalten, die verarbeitet werden können. webtrees enthält Code zur Umgehung dieser Beschränkungen, aber es wurde nicht auf jeder MySQL-Version getestet. Wird sie auf einen Wert größer als Ihre größte GEDCOM-Datei gesetzt, sollte dies Probleme verhindern. 

max-allowed-packet=32M

Auf einem Webserver sind 60 Sekunden eine angemessene Zeit, um auf einen anderen Prozess zum Lösen einer Verriegelung warten. Wenn Sie andere Datenbank-Anwendungen ausführen, benötigen Sie evtl. höhere/niedrigere Werte. 

wait-timeout=60

Obwohl MySQL in erster Linie ein Netzwerk-basierter Server ist, können Sie mit ihm auch direkt kommunizieren, über eine 'pipe' (Windows) oder einen 'socket' (Linux). Hierfür stellen Sie den Servernamen auf "." anstelle von "localhost" oder "127.0.0.1". Das umgeht das Netzwerkprotokoll, was zu einer etwas besseren Performance führt. 

# For Windows
socket=MySQL
enable-named-pipe=1

# For Linux
#socket=/var/run/mysqld/mysql.sock
#bind-address=127.0.0.1

Protokollierung

log-error=mysql.err

Das allgemeine Protokoll enthält jede Abfrage, die MySQL ausführt. Entwickler können es aktivieren, um damit spezifische Probleme zu untersuchen. 

general-log=0

Das Protokoll für langsame Abfragen (wie der Name schon sagt) ist ein Protokoll aller Abfragen, die länger als eine bestimmte Anzahl von Sekunden laufen. Entwickler können es aktivieren, um damit Performance-Probleme zu untersuchen. Die Option, Abfragen ohne Benutzung von Indizes zu protokollieren, kann nützlich sein, aber denken Sie daran, dass für kleine Tabellen das Durchsuchen der ganzen Tabelle schneller sein kann als Index-Zugriff. 

# MySQL 5.1.29 and later
slow-query-log=0

# MySQL 5.1.28 and earlier
#log-slow-queries=1

log-queries-not-using-indexes=0
long-query-time=1

Leistung und Caching (zwischenspeichern)

Diese Einstellungen stellen einen vernünftigen Wert für Arbeitsspeicher zur Verarbeitung von MySQL Abfragen dar. 

# MySQL 5.1.3 and later
table-open-cache=1024

# MySQL 5.1.2 and earlier
table-cache=1024

key-buffer-size=64M
sort-buffer-size=16M

Diese Option speichert das Ergebnis jeder Abfrage, die Sie ausführen, so dass, wenn Sie sie wieder ausführen, die im Cache gespeicherten Werte verwendet werden. Diesen Wert zu hoch zu setzen, kann kontraproduktiv sein. 

query-cache-size=64M

InnoDB Einstellungen

InnoDB ist eine Datenbank-Speicher-Maschine, die viele Vorteile gegenüber der Standard MyISAM-Maschine bietet. webtrees benutzt sie, wenn sie verfügbar ist, und greift auf MyISAM zurück, wenn sie nicht verfügbar ist.

Die Einstellung für innoDB-Log-file-size sollte den Größen der Dateien ib_logfile0 und ib_logfile1 entsprechen, die im Ordner der unter datadir angegeben Einstellung liegen. Wenn Sie diese Einstellung ändern wollen und diese Dateien existieren (sie werden, wenn Sie InnoDB-Tabellen verwendet haben), dann müssen Sie die folgenden Schritte durchführen;

  1. herunterfahren, neustarten, MySQL herunterfahren; das stellt sicher, dass alle Daten auf die Festplatte geschrieben werden
  2. diese beiden Dateien löschen (verschieben in einen temporären Ordner!)
  3. MySQL starten; das erstellt die Dateien neu
  4. Wenn MySQL startet und Sie auf Ihre MySQL-Tabellen zugreifen können, löschen Sie dann die alten Dateien aus Schritt 2
  5. Wenn MySQL nicht startet, stellen Sie sowohl Ihre ursprüngliche Datei my.ini und diese beiden Dateien wieder her. MySQL sollte jetzt starten und Sie können die Datei mysql.err überprüfen, um herauszufinden, welche Probleme gefunden wurden.
innodb=1
innodb-data-file-path=ibdata1:32M:autoextend
innodb-buffer-pool-size=256M
innodb-log-file-size=64M
innodb-log-buffer-size=8M
innodb-fast-shutdown=0
innodb-rollback-on-timeout=1

Standardmäßig überträgt MySQL jede InnoDB-Transaktion auf die Festplatte. Für einige Operationen (z.B. Import einer GEDCOM-Datei) ist dies ein großer Engpass. Durch Setzen des Wertes auf 0 überträgt MySQL nur jede Sekunde auf die Festplatte. Dies macht den Import viel schneller, aber der Nachteil ist, dass ein Server-Ausfall zu einem Verlust der letzten Sekunde führen kann. Wenn Sie andere (vielleicht kritischere) Datenbank-Anwendungen betreiben, dann belassen Sie den Standardwert von 1. 

innodb-flush-log-at-trx-commit=0

Bei sehr langsamen Systemen (z.B. Netbooks mit Solid State Disks), entfernen Sie die Kommentare für zusätzliche Leistung. Sie entfernen bestimmte Kontrollen und Platten-Schreibzugriffe, die nur wichtig werden, wenn Sie Server-Fehler und Hardware-Ausfälle haben. Auf einer Entwicklungs-Maschine ist Leistung entscheidend, also kommen Sie ohne sie aus. 

#innodb-support-xa=0
#innodb-checksums=0
#innodb-doublewrite=0
