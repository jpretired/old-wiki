Viele Leute haben Probleme beim Hochladen von Medien-Objekten, weil es zu Fehlermeldungen kommt, wie:

Miniaturbild 'media/thumbs/wont_upload.jpg' konnte nicht automatisch erzeugt werden.

Wie unter Häufig_gestellte_Fragen erklärt, gilt das fast immer bei zu großen Bildern oder zu wenig PHP Speicher. Benutzer haben berichtet, dass manchmal eine Datei (sagen wir 850 kB groß) hochgeladen wird, aber eine andere, etwas kleinere (etwa 700 kB) nicht. Hier möchte ich erklären, wie es dazu kommt.

Der Grund für die Verwirrung ist, dass die Größe der betreffenden Datei VOR der Komprimierung entscheidend ist (in der Regel ein als jpeg formatiertes Bild) und NICHT die Größe danach. Erläuterung: Um eine angemessene Qualität für Miniaturbild-Kopien zu erhalten, muss PHP zunächst das hochgeladene Bild in seiner unkomprimierten Größe komplett in den internen Server-Speicher laden, bevor es dann auf die viel kleinere Vorschau-Größe reduziert werden kann.

Da sich Bilder je nach Inhalt sehr unterschiedlich komprimieren lassen, kann ein mit 700 KB gespeichertes Bild ohne weiteres größer sein als ein mit 850KB gespeichertes. Die meisten Scanner und Digitalkameras komprimieren Dateien automatisch beim Erstellen, deshalb bedeutet "Komprimieren" nicht, dass Sie oder Ihre Verwandten etwas Ungewöhnliches getan haben müssen.

Falls Sie Probleme haben, dann können Sie die ECHTE Größe Ihrer Bilder in etwa wie folgt berechnen:

Stellen Sie zunächst die Pixelgröße des Bildes fest. Das kann in vielfältiger Weise geschehen und wird (oder kann) oft vom Dateisystem auf Ihrem PC angezeigt werden, wie hier von einem System mit Windows 7:


Beachten Sie, dass die Abmessungen der größeren (875 kB) Datei tatsächlich weit kleiner sind als die der kleineren (718 kB) Datei, was auf sehr unterschiedliche Komprimierung schließen läßt.


Als nächstes berechnen Sie die Anzahl Pixel (px) pro Datei:

will_upload.jpg = 863 x 1176 = 1.014.888 px

wont_upload.jpg = 3436 x 2410 = 8.280.760 px


Um sie in Datei-Größen zu konvertieren, müssen Sie die "Farbtiefe" oder Anzahl Farben kennen, mit der dieses Bild erstellt wurde. Dies ist in der Regel in der Bildinformation gespeichert und kann mit einer Bildbearbeitungs-Software angezeigt werden. In diesen Beispielen sind beide als "RGB" (256 Farben) oder mit 8 Bit pro Kanal angelegt. Die Tabelle unten zeigt einige gängige Alternativen. Die Tiefe (8 in diesem Fall) geteilt durch 8 ergibt die Anzahl "Bytes" pro Pixel:

8 / 8 = 1 Bytes pro Pixel

(24 / 8 = 3 Bytes pro Pixel, etc.)


Und daher für diese Bilder:

will_upload.jpg = 1.014.888 x 1 = 1.014.888 Bytes

wont_upload.jpg = 8.280.760 x 1 = 8.280.760 Bytes


In einem kB sind 1,024 Bytes, also war die echte ORIGINAL-Größe von jedem Bild:

will_upload.jpg = 1.014.888 / 1.024 = 991 kB (gerade unter 1 MB)

wont_upload.jpg = 8.280.760 / 1.024 = 8.087 kB (gerade über 8 MB)


Hoffentlich wird damit klar, warum das "größere" Bild erfolgreich hochgeladen werden konnte und das "kleinere" nicht. Die kleinere Datei war eigentlich 8 mal größer!

Wenn Sie erwägen, Dateien in webtrees zu laden, ist es sinnvoll, über alle folgenden Punkte nachzudenken:

  • webtrees ein Programm für Familiengeschichte und KEINE Foto-Galerie. Bilder in archivarischer Qualität werden weder benötigt, noch gut funktionieren, falls sie benutzt werden.
  • Nicht alle Benutzer werden Ultra-High-Speed-Internet-Verbindungen haben. Die Anzeige eines großen Bildes in Archiv-Qualität könnte das Laden einer Seite im Extremfall für einige Minuten verzögern.
  • Verringern Sie die Auflösung der Bilder auf rund 72 Pixel/Inch. Dies ist in der Regel ausreichend für Webseiten-Darstellung.
  • Die Bildmaße sollten für die meisten Computer-Bildschirme angemessen sein. Etwa nicht breiter als 1200 Pixel ist wahrscheinlich mehr als ausreichend.
  • Wandeln Sie Bilder in schwarz/weiß (1 Bit) um, wo das möglich ist. Viele alte Dokumente wurden nur in schwarz/weiß produziert, also ist hohe Farbtiefe sinnlos.
  • Verwenden Sie Bildbearbeitungs-Software, um Bilder auf benötige Teile zu beschneiden. Das Bild sieht besser aus und verringert die Speicher-Anforderungen.
