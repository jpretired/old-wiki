Hinzufügen von Multimedia-Dateien (MM) zu Ihrem Stammbaum ist eine sehr schöne Funktion, die Ihre Familiengeschichte lebendig macht.

Von der mehr wissenschaftlichen Perspektive können MM als Dokumentation verwendet werden. Sie können Scans von Quellen, Fotos von Grabsteinen oder Audio-Interviews mit Informanten, etc. bereitstellen.

Medientypen

Multimedia-Formate gibt es viele. Obwohl webtrees mit den meisten davon klar kommt, gibt es einige allgemeine Dinge zu beachten.

  • Formate Bilder können in vielen Formaten bearbeitet und gespeichert werden. Die gängigsten Formate sind .jpg, .png, .bmp, .gif, etc.. Sie alle haben ihre Stärken und Schwächen, vor allem in Bezug auf Dateigröße und Genauigkeit. Generell werden .jpg-Bilder als besonders effizient in Bezug auf Speicherplatz angesehen. Das liegt daran, dass eine .jpg-Datei eine verlustbehaftete Komprimierung verwendet, um die Dateigröße zu verringern. Für Fotos auf einer Webseite ist dies in der Regel mehr als ausreichend. Diagramme und Zeichnungen werden jedoch am besten im .png-Format erstellt. Das .png-Format ist auch verlustfrei und kann mit Transparenz arbeiten.
  • Bildgröße Alle Bilder speichern Messinformationen auf zwei verschiedene Arten: Pixelmaße und Auflösung. Ein Pixel ('Bild Element') ist der kleinste Baustein eines digitalen Bildes und entspricht den Punkten auf dem Bildschirm. Wie groß also ein Bild auf dem Bildschirm angezeigt wird, hängt von der Bildschirmauflösung ab. Die meisten modernen Computer-Bildschirme haben 1024x768 Pixel oder mehr, aber nicht der ganze Platz steht zum Betrachten von Bildern zur Verfügung. Falls Ihre Bilder vor allem der Bildschirm-Anzeige dienen und weit über diese Größe hinausgehen, ist das wahrscheinlich Verschwendung, da Sie entweder nicht das ganze Bild auf dem Bildschirm sehen oder das Bild in Ihrem Browser verkleinert wird.
  • Auflösung Die Auflösung eines Bildes wird in der Regel in "dpi" (dots/inch) angegeben, aber das ist nur für gedruckte Bilder relevant. Bei der Betrachtung der Bilder auf dem Bildschirm ist der einzig richtige Weg Gesamtbildpunkte oder Pixel zu verwenden. Zum Drucken kann ein Bild je nach Drucker eine Auflösung von 150-300 dpi oder mehr haben, während Bildschirm-Auflösungen selten mehr als 50 Pixel pro Zoll haben. Wenn Ihr Bild nie gedruckt wird, können Sie die Auflösung (und damit die Dateigröße) sicher senken, ohne die Bildqualität zu beeinflussen. Wenn jedoch ein Bild mit niedriger Auflösung mit zu großer Vergrößerung gedruckt wird, wird die Qualität leiden; es wird ein körniges - pixeliges - Aussehen haben. Wenn Sie also erwägen, Ihre Bilder zu drucken, sollten Sie eine Auflösung von 150 dpi oder mehr verwenden. Ansonsten ist 72 dpi in der Regel ausreichend.
  • Farbtiefe Für jedes Pixel in einem Bild wird ein Farbwert gespeichert (.jpg-Bilder speichern jedoch nur Durchschnittswerte für Pixelbereiche, was zur Reduzierung der Dateigröße führt). Die Genauigkeit dieser Farbinformation kann von reinem Schwarz-Weiß (zwei Farben) zu "true colors" (Millionen Farbnuancen) variieren. Je genauer die Farben, desto größer die Dateigröße.
  • Dateigröße Zusammengenommen bestimmen die aufgeführten Parameter die Dateigröße des Bildes. Wenn dasselbe Original-Bild verwendet würde, um jedes der Formate zu erstellen, wird das angezeigte Bild in gleicher Größe erscheinen, egal welches Format verwendet wird. Allerdings werden sich die Bilddateien in der Größe beträchtlich unterscheiden. Je größer das Originalbild in Bezug auf Pixelgröße und Farbtiefe ist, desto größer wird die resultierende Dateigröße sein.

Bandbreite/Größenbetrachtungen

Warum ist es wichtig, die Dateigröße gering zu halten?

  • Ladezeit. Wenn Sie große Dateien haben, müssen Benutzer - einschließlich Sie selber - lange auf das Laden der Seite vom Server warten. Nicht jeder ist mit Kabelanschluss, Breitband- oder DSL gesegnet.
  • Ihr Webspace ist begrenzt. Je größer Ihre Dateien und mehr große Dateien vorhanden sind, desto mehr Webspace benötigen Sie auf dem Server. Je mehr Platz Sie benötigen, desto höher sind Ihre Kosten.
  • Bandbreite. Je mehr Daten Ihr Server an den Browser des Benutzers senden muss, desto mehr müssen Sie bezahlen. Sie haben möglicherweise ein Hosting-Paket, so dass Traffic kein Problem für Sie ist. Dennoch ist es gut, die Menge unnötiger Daten zu verringern, die über das Internet hin und her gesendet werden. Wenn eine 20kB Datei ausreichend ist, warum dann 1 MB ver(sch)wenden?


Hochladen und Verwalten von Medien

Es gibt mehrere Möglichkeiten, Medien auf Ihre Webseite hochzuladen, vorausgesetzt Sie haben Bearbeitungsrechte.

Im Benutzerbereich

Die häufigste Methode bei täglicher Nutzung ist, den Link "Ein neues Medienobjekt hinzufügen" in der Liste der verfügbaren Extras am unteren Ende aller Bearbeitungsfenster für Fakten, Ereignisse, Quellen, usw. oder auf der Registerkarte "Multimedia" des Personen-Datensatzes zu verwenden.

Das Verfahren ist weitgehend selbsterklärend: In dem Dialog-Fenster können Sie eine Datei zum Hochladen von Ihrem Computer auswählen (für andere Möglichkeiten s. Entfernte Medien). Wenn Sie wollen, können Sie auch ein Miniaturbild hochladen (wenn nicht, wird es automatisch erzeugt). Andere Dateinamen auf dem Server, einen Bild-Titel und ob dieses Bild als Hauptbild für die Person, der es zugeordnet ist, verwendet werden soll, können Sie auch angeben. Diese zusätzlichen Einstellungen sind optional. Es wird jedoch empfohlen, dass Sie mindestens einen Titel angeben, andernfalls wird der Dateiname verwendet.

Im Verwaltungsbereich

Im Medien-Management (Verwaltung -> Multimedia) können Sie "Multimediadateien hochladen" auswählen und ein Element laden und zugleich mit einer Person verknüpfen.

Wenn Sie mehrere Medienobjekte hochladen müssen, können Sie den Dialog "Multimediadateien hochladen" verwenden. Dies ermöglicht bis zu 5 Elemente auf einmal hochzuladen. Diese müssen dann jeweils mit einer Person, Familie, Ereignis oder anderen Datensätzen verknüpft werden, nachdem der Upload abgeschlossen ist.

Wenn Sie eine große Anzahl von Medien-Objekten zum Hochladen haben, sollten Sie den Administrator kontaktieren, um die besten Möglichkeiten zu diskutieren.


Verwalten Sie Ihre Mediendateien

Im Benutzerbereich können Sie die Details einzelner Dateien bearbeiten. Die systematischere Bearbeitung mehrerer Dateien wird bequemer auf der Medienseite im Verwaltungsbereich durchgeführt.

Durch Anwendung von Filtern (alternativ Anzeige von allen) erhalten Sie eine Liste der Bilder und anderen Mediendateien auf der Webseite. Neben jedem Bild in der Medienliste finden Sie eine Reihe von Bearbeitungsmöglichkeiten. Welche Optionen angezeigt werden, hängt vom aktuellen Status der fraglichen Mediendatei ab.

  • Bearbeiten Hier können Sie den Titel des Medienobjekts ändern oder die Datei umbenennen. Wenn das Medienobjekt noch nicht mit einer Person, Familie oder Quelle in der gerade aktiven Datenbank verknüpft ist, können Sie diese Verknüpfung hier herstellen. Sie können auch den Speicherort der Datei in der Verzeichnisstruktur, wo Ihre Mediendateien liegen, ändern. Wenn nötig, wird webtrees automatisch die erforderlichen Unterverzeichnisse oder fehlenden Miniaturbilder erzeugen.
  • GEDCOM Rohdaten bearbeiten Diese Option ist nur verfügbar, wenn der Administrator sie freigegeben hat. Sie sollten sehr vorsichtig sein, wenn Sie diese Option verwenden.
  • Datei löschen Mit dieser Option können Sie jegliche Information über die Mediendatei aus der aktuellen Datenbank löschen. Andere Datenbanken sind nicht betroffen. Wenn die Mediendatei mit keiner anderen Datenbank verbunden ist, werden die Datei und das zugehörige Miniaturbild gelöscht.
  • Objekt entfernen Mit dieser Option können Sie jegliche Information über das Medienobjekt aus der aktuellen Datenbank löschen. Andere Datenbanken sind nicht betroffen. Das Medienobjekt und das zugehörige Miniaturbild werden nicht gelöscht.
  • Verknüpfungen entfernen Mit dieser Option können Sie jegliche Verknüpfungen mit dem Medienobjekt aus der aktuellen Datenbank löschen. Andere Datenbanken sind nicht betroffen. Die Datei wird nicht gelöscht und das Medienobjekt, unter dem diese Datei in der aktuellen Datenbank bekannt ist, wird beibehalten.
  • Verknüpfung setzen Mit dieser Option können Sie Verknüpfungen zwischen Mediendateien und Personen, Familien oder Quellen in der aktuellen Datenbank herstellen. Wenn nötig, wird webtrees auch das Medienobjekt erstellen, unter dem die Mediendatei in der Datenbank bekannt ist.
  • Miniaturbild erstellen Wenn Sie diese Option wählen, wird webtrees das fehlende Miniaturbild erstellen.


Zuordnen von vorhandenen Medien zu Menschen oder Quellen

Jedes Medien-Element sollte mit einer oder mehreren Personen-, Familien- oder Quellen-Datensätzen in der Datenbank verbunden sein.

Um eine solche Verknüpfung herzustellen, können Sie die ID der Person, Familie, oder die Quelle zur gleichen Zeit, zu der Sie das Medien-Objekt erstellen, eingeben oder suchen. Sie können die Verknüpfung aber auch später mit Hilfe der Bearbeitungsmöglichkeiten im Multimedia-Dialog herstellen oder durch Hinzufügen von Medienobjekten mit dem Link "Ein neues Medienobjekt hinzufügen" auf den Personen-, Familien- oder Quellen-Detailseiten.


Unterverzeichnisse einbeziehen

Wenn diese Option gewählt ist, werden die Multimedia-Objekte entsprechend der Filterliste nicht nur im aktuellen Verzeichnis, sondern auch allen seinen Unterverzeichnissen gesucht. Wenn die Option "Unterverzeichnisse ausschließen" gewählt ist, wird nur das ausgewählte Verzeichnis durchsucht.

Die Titel aller gefundenen Medienobjekte werden dann untersucht, um festzustellen, ob der eingegebene Text im Filter enthalten ist. Das Ergebnis dieser beiden Maßnahmen bestimmt die aufgelisteten Multimedia-Objekte.

Bildanzeige ("Album")

Die Medien-Anzeige kann den Namen der angezeigten Medien-Datei ausgeben. Die Option "Diabetrachter erlauben, Download-Links zu zeigen" legt fest, ob der Dateiname für Benutzer gezeigt wird oder nicht.

Sie wollen den Dateinamen aus Sicherheitsgründen eventuell unterdrücken.

Miniaturbilder

webtrees nutzt Miniaturbilder als Vorschau auf Bilder und auf der Seite des Personen-Datensatzes sowie Diagrammen. Standardmäßig werden die Vorschaubilder automatisch erzeugt, aber Sie können wählen, Ihre eigenen Vorschaubilder zu generieren.

In jedem neuen Medien-Unterverzeichnis, das Sie erstellen, wird webtrees auch ein Unterverzeichnis thumbs mit identischer Verzeichnisstruktur wie das Verzeichnis selbst erzeugen.

Wenn Sie ein Bild oder eine andere Art von Mediendatei hochladen, gibt es eine Reihe von Optionen mit Bezug zu Vorschaubildern.

  • Automatisch oder Manuell? Für die gängigsten Bildformate (JPEG, PNG, GIF) kann webtrees Miniaturbilder automatisch erzeugen.
    Sie wollen eventuell stattdessen eigene Vorschaubilder manuell generieren. Für viele Formate wird dies die einzige Option sein. Zum Beispiel können Sie ein Standbild aus einem Video oder ein Foto einer Person, die eine Audio-Aufnahme gemacht hat, beistellen.
    Vorschaubilder werden in einer parallelen Verzeichnisstruktur im Medienverzeichnis gespeichert, wo sie den gleichen Namen wie das Hauptbild erhalten.
  • Hervorgehobenes Bild? Sie können wählen, welche Mediendatei als Haupt-Bild einer Person, das auf der Seite des Personen-Datensatzes und Diagrammen angezeigt wird, verwendet werden soll. Wenn verfügbar, wird die Miniaturansicht verwendet.


Zufallsbilder aus dem aktuellen Stammbaum

Die Einstellung "Anzeige der Steuerungselemente der Diaschau" können Sie verwenden, um die Bedienelemente der Zufallsbilder im Block Diaschau anzuzeigen oder auszublenden.

Diese Bedienelemente ermöglichen dem Benutzer, auf ein anderes Zufalls-Objekt zu springen oder wie in einer Diaschau zufällig ausgewählte Medien anzuzeigen. Die Diaschau ändert den Inhalt des Blocks ohne vorheriges Laden der Informationen vom Server und ohne Nachladen der gesamten Seite.

Sie können beschränken, was die Diaschau je nach Format und Typ der Medien-Objekte anzeigen darf. Je nach aktiviertem Kontrollkästchen darf die Zufallsanzeige das Medien-Element dieses Formats oder Typs anzeigen.

Format oder Typ-Kennzeichen, die in der Datenbank existieren, aber nicht in dieser Auswahlliste, werden als aktiviert angenommen. Zum Beispiel, wenn Ihre Datenbank Medienobjekte vom Format pdf enthält, darf die Diaschau sie immer anzeigen. Und falls Ihre Datenbank Medienobjekte vom Typ spezial enthält, darf die Diaschau sie immer anzeigen.

Mit der Option "Personen, Ereignisse oder alle Medien zeigen?" können Sie die Art der Medien für die Anzeige bestimmen.

Wenn Sie Personen auswählen, werden nur Medien, die mit Personen verbunden sind, angezeigt. Normalerweise wäre dies das Photo einer Person. Wenn Sie Fakten und Ereignisse wählen, werden nur Medien, die mit Fakten oder Ereignissen verbunden sind, angezeigt. Dies könnte ein Bild einer Urkunde sein. Wenn Sie Alles wählen, zeigt der Block alle Medien-Arten an.


Medien-Speicherung

Medien, die Ihrer Webseite hinzugefügt werden, werden in einem Verzeichnis oder Verzeichnisbaum auf dem Server gespeichert. Mediendateien werden hinter der "Medien Firewall" in einem Server-Verzeichnis, das nicht aus dem Internet erreichbar ist, gespeichert.

Sie können wählen, ob ein einzelnes Verzeichnis für alle Mediendateien oder eine Verzeichnisstruktur mit Unterverzeichnissen verwendet werden soll.

Dateinamen

Lassen Sie das Feld Dateiname leer, um den ursprünglichen Namen der Datei, die Sie von Ihrem lokalen Computer hochgeladen haben, zu erhalten.

Die Mediendatei, die Sie hochladen, kann und wahrscheinlich sollte auf dem Server anders benannt sein, als auf dem lokalen Computer. Dies ist so, weil lokale Dateinamen für Sie oft eine Bedeutung haben, die aber anderen Besuchern dieser Webseite viel weniger sagen. Beachten Sie auch die Möglichkeit, dass Sie und jemand anderes versuchen könnte, verschiedene Dateien beide mit dem Namen "oma.jpg" hochzuladen.

Verzeichnis-Struktur

Der Weg zu einem lesbaren und beschreibbaren Verzeichnis, in dem webtrees Mediendateien (einschließlich des abschließenden "/") ablegt, muss nicht auf den Namen "Medien" lauten. Sie können einen beliebigen Namen wählen.

Auch wenn die Medien-Firewall-Funktion Sie Mediendateien in einem Dateibereich des Servers speichern lässt, der nicht aus dem Internet zugänglich ist, muss das Verzeichnis dennoch bestehen und muss aus dem Internet lesbar und für webtrees beschreibbar sein. Für weitere Informationen, wird auf die Konfigurationsoptionen der Medien-Firewall in der Kategorie Systemverwaltung verwiesen.


Entfernte Medien

Medien-Dateien müssen nicht auf dem Server gespeichert werden. Anstelle der Dateiauswahl im Feld Multimedia Datei zum Hochladen können Sie eine gültige Internet-Adresse zu einer Datei an einem anderen Server eingeben, z.B. http://niceandfreepics.org/old-man.jpg. Beachten Sie, dass, außer Sie haben ein Miniaturbild bestimmt, die gleichen Nachteile, die oben unter Miniaturbilder diskutiert wurden, gelten: ein volles Bild von einem entfernten Server muss für Diagramme erst angefordert werden und kann, wo das Bild angezeigt wird, zu langsamem Seitenaufbau führen. Ein weiterer Nachteil ist, wenn die Datei auf dem entfernten Server gelöscht oder verschoben wird, dass sie natürlich auch auf Ihrer Website verschwindet.

ACHTUNG: Sie sollten sicherstellen, dass Sie die Rechte haben, das Bild zu verwenden!

Auf der Registerkarte Multimedia der Stammbaum-Einstellungen können Sie wählen, entfernte Links zu bewahren. Wenn ein Multimedia-Link gefunden wird beginnend mit beispielsweise http://, ftp://, mms:// wird der Link nicht verändert, wenn die Option auf Ja gesetzt ist. Zum Beispiel bleibt http://www.myfamily.com/photo/dad.jpg als http://www.myfamily.com/photo/dad.jpg erhalten. Wenn die Option auf Nein gesetzt ist, wird der Link als Standard-Referenz behandelt und die Medien Tiefe verwendet. Zum Beispiel: http://www.myfamily.com/photo/dad.jpg wird zu /media/dad.jpg.

Dies hat keine Auswirkungen darauf, wie das Bild selbst angezeigt wird, nur auf die Adresse, die auf das Bild verweist. Somit wird, wenn die Seite angezeigt wird, die Datei http://www.myfamily.com/photo/dad.jpg von dem entfernten Server geholt, aber ein Klick den Link führt Sie zu /media/dad.jpg. Dies ist wahrscheinlich nur dann eine nützliche Option, wenn http://www.myfamily.com die Adresse Ihrer webtrees Installation auf Ihrem eigenen Server und das Foto-Verzeichnis mit dem Medienverzeichnis verknüpft ist, andernfalls ist es ein toter Link.

Wasserzeichen

Wenn die Medien-Firewall aktiviert ist, sollten Kopien von mit Wasserzeichen versehenen Bildern in voller Größe auf dem Server zusätzlich zu den gleichen Bildern ohne Wasserzeichen gespeichert werden?

Wenn Ja gewählt wird, werden Bilder in voller Größe mit Wasserzeichen schneller, jedoch auf Kosten von höheren Server-Speicherplatz-Anforderungen erstellt.


Exportieren und Herunterladen von Medien

Die Anzeige der Medienobjekte kann einen Link anzeigen, der, wenn darauf geklickt wird, die Medien-Datei auf einen lokalen PC herunterladen kann.

Sie können den Link zum Herunterladen aus Sicherheitsgründen verbergen.

Diese Option bestimmt, ob die Verzeichnisnamen in der Angabe FILE der Medienobjekte durch Schrägstriche oder Rückwärtsschrägstriche voneinander getrennt werden. Ihre Wahl hängt von den Anforderungen des empfangenden Betriebssystems ab.

Die Wahl Schrägstriche: / passt für die meisten Betriebssysteme außer Microsoft Windows. Die Wahl Rückwärtsschrägstriche: \ sollte verwendet werden, wenn das Zielprogramm auf einem Microsoft Windows-System ausgeführt wird.

Medienpfade, die eigentlich Internetadressen sind, werden nicht geändert.

Wenn Sie Medienobjekte in webtrees erstellt und Ihre GEDCOM offline mit einem Programm bearbeitet haben, das Medienobjekte löscht, dann wählen Sie dieses Feld, um die aktuellen Medienobjekte mit der neuen GEDCOM zu verschmelzen.
