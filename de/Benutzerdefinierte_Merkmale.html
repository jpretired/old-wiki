Benutzerdefinierte Merkmale

Aus webtrees
Wechseln zu: Navigation, Suche

Viele Anwender, vor allem unerfahrene Anwender versuchen, die GEDCOM-Spezifikation mit eigenen Merkmalen zu ergänzen. Diese Seite tut drei Dinge:

  1. Sie versucht, Sie davon abzuhalten.
  2. Sie zeigt Ihnen Alternativen.
  3. Sie erklärt, wie man es machen kann. Dies nicht, weil wir diese Technik unterstützen, fördern oder billigen. Es erspart uns einfach, es wieder und wieder zu erklären. Wahrscheinlich gibt es andere Wege, das zu tun, dies ist nur einer.

Gründe benutzerdefinierte Merkmale zu vermeiden

Es gibt viele Gründe, benutzerdefinierte Merkmale zu vermeiden.

  1. Andere Anwendungen werden sie ignorieren (oder sogar löschen).
  2. Bedeutung oder Aussagekraft kann ihnen nicht zugewiesen werden.
  3. Andere Genealogen wird nicht wissen, was sie bedeuten.
  4. Die GEDCOM-Spezifikation rät davon ab.
  5. Sie machen es schwierig, Ihre Daten in andere Anwendungen zu migrieren.
  6. Die webtrees Entwickler denken, es ist eine schlechte Idee. Wir machen es nicht bei uns selbst und wir unterstützen es nicht.

Alternativen zu benutzerdefinierten Merkmalen

Benutzen Sie das "TYPE" Kennzeichen zusammen mit einem bestehenden Merkmal

Professionelle Schauspieler und Entertainer benutzen häufig einen Künstlernamen. Anstelle eines besonderen Merkmals für "Künstlername", kann man folgendes definieren 

1 NAME John /Wayne/
2 TYPE Künstlername

Verwenden Sie ein vorhandenes benutzerdefiniertes Merkmal

Eine Reihe von benutzerdefinierten Merkmalen wurden von verschiedenen Desktop-Anwendungen erstellt und webtrees erkennt viele von ihnen. Anstelle ein Merkmal zu erstellen, um anzuzeigen, dass jemand blind war, könnten Sie ein vorhandens "medizinisches" Merkmal verwenden. 

1 _MDCL Blind von Geburt an

Obwohl webtrees die Bezeichnung dieses Merkmals übersetzen kann, ergeben nur sehr wenige von ihnen einen Sinn. Zum Beispiel die Merkmale für _NMR (nicht verheiratet) und _MARNM (Ehenamen).

Hinweis: Unterstützung für diese Merkmale gibt es nur aus historischen Gründen. Wenn wir nochmal neu anfangen würden, würden wir es wahrscheinlich nicht mehr machen. Stattdessen würden wir sie alle EVEN, FACT oder NOTE Merkmalen zuordnen.

Verwenden Sie eine Bemerkung

Sie können fast alles in einer Bemerkung unterbringen. Anstatt ein Merkmal zu erstellen, dass jemand Analphabet war, könnten Sie eine Bemerkung benutzen: 

 1 NOTE Analphabet - unterschrieb alle Dokumente mit einem X

Aber wenn es denn sein muss ...

Um ein Merkmal _FOOBAR hinzufügen

Ergänzen Sie das Merkmal in der Benutzeroberfläche

Auf der Stammbaum Konfigurations-Seite wählen Sie die Registerkarte "Bearbeiten" und ergänzen Sie _FOOBAR beim entsprechenden Person/Familien/Quellen/etc. Datensatz.

Ergänzen Sie das Merkmal bei der internen Liste

Bearbeiten Sie die Datei library/WT/Gedcom/Tag.php und:

  1. ergänzen Sie _FOOBAR zur Definition von $ALL_TAGS
  2. ergänzen Sie die englische Kennzeichnung für die Variable in der Funktion getLabel() z.B. 'Foo Bar'. Möglicherweise müssen Sie separate Kennzeichen für den Einsatz bei Männern und Frauen erstellen. Kopieren Sie den Code von einem der vorhandenen Merkmale.

Ergänzen Sie Übersetzungen für andere Sprachen

Erstellen Sie eine Übersetzung für 'Foo Bar' in jeder Ihrer unterstützten Sprachen unter Beachtung der Anweisungen in Benutzerdefinierte_Übersetzungen. Nicht übersetzte Sprachen zeigen den englischen Text.

Allgemeine Bemerkungen

  • Beachten Sie, dass Sie die Änderungen an dieser Datei jedes Mal vornehmen müssen, wenn Sie auf eine neuere Version von webtrees updaten. Das Online Update Werkzeug (verfügbar ab Version 1.5.1) erstellt eine Datensicherung dieser Datei als teil des Updates, so dass Ihre Änderungen nicht verloren gehen.
